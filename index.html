Performance-First

Section 04

AI Engine — Gemini 2.5

The intelligence core. How the AI processes, understands, and tutors from examination papers.

Chain-of-Thought Prompting

Structured reasoning for question extraction

Instead of naive "find all questions" prompts, the AI is instructed to identify three distinct components for every question:

Stem

The actual question being asked, including context and constraints.

Options

All answer choices (A/B/C/D) with their full text preserved.

Cognitive Level

Bloom's taxonomy classification (Remember → Create).

Multimodal OCR

Direct image-to-text for formulas & diagrams

Mathematical formulas, chemical equations, circuit diagrams, and complex tables are processed directly as images. Gemini 2.5's native vision capabilities eliminate the need for traditional OCR tools like pdftoppm which had Windows compatibility issues.

Legacy vs Native Copy 
// ❌ Legacy (removed)
const images = await pdfToImages(buffer); // pdftoppm — Windows broken

// ✅ Current (Gemini Native)
const result = await model.generateContent({
  contents: [{
    parts: [
      { inlineData: { mimeType: 'application/pdf', data: base64 } }
    ]
  }]
});

Socratic Context Injection

System instructions for tutoring style

The AI's system instruction enforces a Socratic tutoring style. When a user asks about a question, it doesn't just reveal the answer — it guides through the reasoning:

STEP 1

Identify the underlying concept ("This is a question about Newtonian mechanics...")

STEP 2

Ask a guiding question ("What forces are acting on the object?")

STEP 3

Scaffold the answer progressively based on student responses

Safety Configuration

BLOCK_NONE for educational content

Educational exam content (biology diagrams, historical events, chemical processes) can trigger false-positive safety filters. The safety settings are explicitly configured to prevent this: 

const safetySettings = [
  { category: "HARM_CATEGORY_HARASSMENT", threshold: "BLOCK_NONE" },
  { category: "HARM_CATEGORY_HATE_SPEECH", threshold: "BLOCK_NONE" },
  { category: "HARM_CATEGORY_SEXUALLY_EXPLICIT", threshold: "BLOCK_NONE" },
  { category: "HARM_CATEGORY_DANGEROUS_CONTENT", threshold: "BLOCK_NONE" },
];

Note: This configuration is intentional for a controlled educational environment. In production, consider implementing a custom content filter layer.

Section 05

Study Pulse — Music Engine

An ambient focus music system built with redundancy and modern browser API awareness.

Glassmorphic UI

Top-right translucent controller

A vertical control stack positioned in the top-right corner with backdrop-blur glassmorphism. Shows real-time status diagnostics: current source, play state, and fallback status.

Triple-Redundancy

Automatic CDN source cycling

Three independent CDN sources (SoundHelix, Pixabay, private stream) are maintained. If one fails — due to CORS, rate limiting, or outage — the player automatically cycles to the next without user intervention.

Bug Fix: Pause Button Race Condition

The pause button wasn't working because the global interaction-unlock listener was re-triggering playback on every click. Fixed with e.stopPropagation() on the pause control to prevent event bubbling to the global listener. 

// The fix — stop propagation on pause button
pauseBtn.addEventListener('click', (e) => {
  e.stopPropagation(); // ← prevents global re-trigger
  audio.pause();
});
Section 06

Data Lifecycle

The "Analysis Path" — from PDF upload to interactive tutoring session.

1

Ingestion

User uploads a PDF through the drag-and-drop interface. The frontend converts the file to a Buffer / Base64 string and sends it via POST to the backend API.

2

Transformation

The API server receives the file. Instead of pre-processing with external OCR tools, the raw PDF is sent directly to Gemini 2.5 as a document part — leveraging Gemini's native PDF understanding.

3

Extraction

The model identifies every individual question, generates a unique ID for each, classifies it by subject, and tags the cognitive level (Bloom's taxonomy). Output is structured JSON.

Interaction

User clicks a question in the split-pane viewer. The AI generates a scaffolded answer — explaining the underlying concept, asking guiding questions, and progressively revealing the solution based on student interaction.
